[Leetcode1123]Lowest_Common_Ancestor_of_Deepest_Leaves

Yu' Blog
2019-10-06
algorithm, leetcode

Description

Given a rooted binary tree, return the lowest common ancestor of its deepest leaves.

Recall that:

The node of a binary tree is a leaf if and only if it has no children
The depth of the root of the tree is 0, and if the depth of a node is d, the depth of each of its children is d+1.
The lowest common ancestor of a set S of nodes is the node A with the largest depth such that every node in S is in the subtree with root A.

Example 1:

Input: root = [1,2,3]
Output: [1,2,3]
Explanation: 
The deepest leaves are the nodes with values 2 and 3.
The lowest common ancestor of these leaves is the node with value 1.
The answer returned is a TreeNode object (not an array) with serialization "[1,2,3]".

Example 2:

Input: root = [1,2,3,4]
Output: [4]

Example 3:

Input: root = [1,2,3,4,5]
Output: [2,4,5]

Solution

因为是要求leaves，不难想到要postorder顺序遍历树
一个树的最深的节点可能有多个，如果有多个，则返回的是它们的父亲节点；如果一个树的最深的节点只有一个，则返回的就是节点本身。因此需要构建一个helper函数，并且设置全局变量为 要求的lca节点和最大树高度
Every time we find a deeper leaf, we make this leaf become lca, modify maxDepth, and return its depth.
如果一个结点的左子树和右子树高度相等，那么其左子树的最深结点和右子树的最深结点，都是以这个结点为根的最深叶结点，那么我们就返回这个结点，和这个结点的高度（左子树高度或右子树高度加 1）；
如果一个结点的左子树高度大于右子树，那么以这个结点为根的树，其最深叶结点一定在左子树中，那么我们就返回其左子树中最深结点的最近公共祖先，和当前结点的高度（左子树高度加 1）；
如果一个结点的右子树高度大于左子树，那么我们处理情况和情况2相反，返回右子树中最深结点的最近公共祖先，和当前结点的高度（右子树高度加 1）。

我觉得上面那种把深度和deepest leave高度结合起来的算法不太清晰，可以直接写一个计算某个节点高度的递归函数
